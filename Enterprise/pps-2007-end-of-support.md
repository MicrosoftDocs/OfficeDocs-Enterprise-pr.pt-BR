## <a name="what-does-end-of-support-mean"></a>Qual end faz da média de suporte?

Produtos da Microsoft, como o PerformancePoint Server 2007 SP3, o software ProClarity e o SharePoint Server 2007 SP3 tinham um ciclo de vida de suporte durante os quais a Microsoft fornece novos recursos, correções e atualizações de segurança. O ciclo de vida de um produto geralmente dura por 10 anos a partir da data de lançamento inicial do produto e final do ciclo de vida é conhecido como o fim do produto do suporte. Como o ProClarity, PerformancePoint Server e SharePoint Server 2007 atingiu seu fim do suporte, Microsoft não fornecerá mais:
  
- Suporte técnico para problemas que podem ocorrer
    
- Correções de bug de problemas que são descobertos e que pode afetar a estabilidade e a facilidade de uso de servidores
    
- Correções de segurança para vulnerabilidades que são descobertas e que pode tornar servidores ou aplicativos vulnerável a violações de segurança
    
- Atualizações de fuso horário
    
Sua instalação do ProClarity, SharePoint Server 2007 SP3 e do PerformancePoint Server 2007 SP3 continuará a execução, embora o suporte terminou. No entanto, é altamente recomendável que você migre desses aplicativos assim que possível.

E, caso algo saia errado com os aplicativos do servidor, você não conseguirá Obtenha ajuda de suporte técnico da Microsoft.

### <a name="use-power-bi-in-the-cloud-or-on-premises"></a>Usar o Power BI (na nuvem ou no local)

Power BI é um pacote de ferramentas de análise de negócios para analisar dados e compartilhar ideias. Com o Power BI, você pode criar relatórios interativos e painéis usando fontes de dados online ou local. As pessoas podem exibir e usar seus relatórios e painéis usando seus computadores ou dispositivos móveis.
  
Power BI não está incluído no Office 365 ou no SharePoint Server, mas é um separado oferta que inclui a área de trabalho do Power BI, gateways Power BI e o serviço do Power BI. Power BI também se integra com o SharePoint Online. Você pode começar com Power BI gratuitamente e das suas uso necessidades e dados corporativos, atualizar para o Power BI profissional. Para saber mais, consulte [o que é Power BI?](https://go.microsoft.com/fwlink/?linkid=841341)
  
### <a name="use-reporting-services-on-premises"></a>Usar o Reporting Services (no local)

SQL Server Reporting Services oferece uma solução de relatórios robusta, juntamente com a capacidade de instalar e configurar o Reporting Services no modo nativo ou no modo integrado do SharePoint. Você pode criar relatórios usando várias ferramentas, incluindo Report Designer, construtor de relatórios e Power View. Com a versão mais recente do SQL Server, você também pode usar o Editor de relatório do SQL Server Mobile a fornecer relatórios que se ajustam para qualquer tamanho de tela, oferecendo a capacidade de consumir relatórios em seus dispositivos móveis de sua organização. Para saber mais, consulte [SQL Server Reporting Services (SSRS): criar, implantar e gerenciar relatórios de celular e paginados](https://go.microsoft.com/fwlink/?linkid=841342).
  
### <a name="use-performancepoint-services-on-premises"></a>Usar os Serviços PerformancePoint (no local)

Como você sabe, PerformancePoint Server 2007 foi adquirida separadamente do SharePoint Server 2007. Começando com o SharePoint Server 2010, os Serviços PerformancePoint é um aplicativo de serviço no SharePoint Server. Isso significa que você não precisa a compra de licenças de servidor separado ou hardware para usar os Serviços PerformancePoint.
  
Para mover do PerformancePoint Server 2007 para os Serviços PerformancePoint, você deve move para uma versão mais recente do SharePoint Server e configurar os Serviços PerformancePoint. A versão do SharePoint Server que você está movendo para determinará se você pode importar conteúdo existente do seu painel do PerformancePoint Server 2007 para os Serviços PerformancePoint.
  
- Se você estiver atualizando para o SharePoint Server 2010, você pode importar o conteúdo do PerformancePoint dashboard do PerformancePoint Server 2007 para os Serviços PerformancePoint no SharePoint Server 2010. Para saber mais sobre como isso funciona, consulte [Assistente de importação: conteúdo do PerformancePoint Server 2007 para o SharePoint Server 2010](https://go.microsoft.com/fwlink/?linkid=838873).
    
- Se você estiver movendo para o SharePoint Server 2013 ou SharePoint Server 2016, você provavelmente precisará criar um novo conteúdo de dashboard (fontes de dados, relatórios, scorecards e páginas de painel).
    
Para começar a no seu plano de atualização de Serviços PerformancePoint, consulte os seguintes recursos:
  
1. [Fim do SharePoint Server 2007 do mapa de suporte](sharepoint-2007-end-of-support.md)
    
2. Quando você sabe qual versão do SharePoint que você está movendo para, consulte o artigo correspondente dos Serviços PerformancePoint:
    
  - [Planejar os Serviços do PerformancePoint (SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841363)
    
  - [Visão geral do PerformancePoint Services no SharePoint Server 2013](https://go.microsoft.com/fwlink/?linkid=841551)
    
  - [Visão geral dos Serviços PerformancePoint no SharePoint Server 2016](https://go.microsoft.com/fwlink/?linkid=874704)
    
Quando você faz uma atualização para os Serviços PerformancePoint, você ficará vários novos recursos e melhorias. Serviços PerformancePoint oferece melhores scorecards, novas visualizações, como a árvore de Decomposição e relatório de detalhes de KPI e os tipos de gráfico mais, melhores recursos de filtragem de inteligência de dados temporais e conformidade de acessibilidade aperfeiçoada. Para saber mais, consulte [What's new for PerformancePoint Services (SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841343).
